IMPLEMENTED ENDPOINTS:
=====================

1. POST /api/videos
   - Submit a video URL for processing
   - Creates video record in database
   - Creates transcription task with external ID mapping
   - Enqueues task to Redis queue
   - Request validation: video_url (required, URL format), title (required), duration (required, min=1)
   - Optional: bilibili_id, model_name, priority

2. GET /api/videos
   - List all videos with pagination
   - Query parameters: status (filter), limit (default: 20), offset (default: 0)
   - Returns: videos array, total count, limit, offset

3. GET /api/videos/:id
   - Get video processing status by task ID (UUID)
   - Retrieves task from Redis queue
   - Returns: task_id, video_url, status, model_name, priority, created_at

4. GET /api/videos/:id/transcript
   - Get transcript result for a processed video
   - Maps external task ID to database task ID
   - Retrieves transcript segments from PostgreSQL
   - Returns: transcript with segments (start, end, text)

5. GET /api/videos/:id/summary
   - Get summary result for a processed video
   - Returns: summary text and key points (when available)

6. GET /api/video/:id
   - Get video details by database ID
   - Returns: video info, transcription tasks, summary

DATABASE INTEGRATION:
====================
- PostgreSQL models: Video, TranscriptionTask, TranscriptSegment, VideoSummary
- External ID mapping: UUID task ID stored in TranscriptionTask.external_id
- Transaction support for video + task creation
- Auto-migration on startup

REDIS QUEUE INTEGRATION:
=======================
- Priority-based task queue using sorted sets
- Task states: pending, processing, completed, failed
- Status tracking across multiple queues

ERROR HANDLING:
==============
- Request validation with descriptive error messages
- Database transaction rollback on failure
- Proper HTTP status codes: 200, 400, 404, 500
- Graceful degradation when DB/Redis unavailable
